# 好卡流量卡推荐

## 目的

从 haokawx.lot-ml.com 鸣日科技店铺抓取全部流量卡数据，根据用户的具体需求筛选推荐最匹配的套餐。覆盖电信、联通、移动、广电四大运营商，76+张卡片。

## 使用时机

当用户表达以下需求时触发：
- "推荐一张流量卡"
- "有没有便宜的大流量卡"
- "广东能用的电信卡有哪些"
- "我需要一张月租30以内、流量100G以上的卡"
- "帮我找一张能选号的移动卡"
- 任何涉及手机流量卡/套餐推荐的问题

## 数据源

### 主数据源：agent-browser 实时抓取（优先）

店铺 ID: `8eabe44066833227`
店铺名称: 鸣日科技流量卡
页面地址: `https://haokawx.lot-ml.com/ProductEn/Index/8eabe44066833227`

用 agent-browser 打开页面，等待 JS 渲染完成，用 snapshot 获取完整产品列表数据。这是唯一可靠的实时数据获取方式（urllib/API 直连因缺少浏览器 session 经常返回空数据）。

## 工作流程

### 第一步：获取卡片数据（实时优先，缓存兜底）

**抓取策略：实时优先 + 最多重试3次 + 缓存兜底**

```
尝试1: agent-browser open → wait → snapshot → 解析
  ↓ 失败
尝试2: agent-browser open → wait(更长) → snapshot → 解析
  ↓ 失败
尝试3: agent-browser open → wait → snapshot → 解析（最后一次）
  ↓ 失败
回退缓存: 读取 assets/cards_cache.json + 告知用户数据非最新
```

**实时抓取命令序列**：
```bash
# 第1次尝试
agent-browser open 'https://haokawx.lot-ml.com/ProductEn/Index/8eabe44066833227'
agent-browser wait --load networkidle
agent-browser snapshot

# 如果第1次失败（数据为空或解析失败），第2次尝试
agent-browser wait --load load
agent-browser snapshot

# 如果第2次也失败，第3次尝试（重新打开页面）
agent-browser open 'https://haokawx.lot-ml.com/ProductEn/Index/8eabe44066833227'
agent-browser wait --load networkidle
agent-browser snapshot

# 完成后必须关闭浏览器
agent-browser close
```

**解析 snapshot 数据**：
使用 `scripts/scraper.py` 中的 `parse_snapshot()` 函数，从 snapshot 文本中用状态机提取结构化卡片数据。

**判断"抓取成功"的标准**：
- 解析出 ≥10 张卡片（页面至少有这么多产品）
- 每张卡片有 name 字段（包含【和】）
- 数据总量明显多于0

**使用缓存**：
仅在3次实时抓取全部失败时，读取 `assets/cards_cache.json`，并在推荐结果开头标注：
> ⚠️ 当前使用缓存数据，卡片信息可能不是最新版本。如需最新数据请稍后重试。

### 第二步：理解用户需求

从用户提问中提取筛选条件：

| 维度 | 提取关键词 | 说明 |
|------|----------|------|
| 运营商 | 电信/联通/移动/广电 | 单选或多选 |
| 省份 | 广东/全国/北京/上海等 | 部分卡限制发卡区域 |
| 城市 | 广州/深圳等 | 部分卡仅限特定城市 |
| 月租预算 | 30以内/50以内 | 上限金额 |
| 流量需求 | 100G以上/200G | 通用流量下限 |
| 通话需求 | 需要通话/100分钟以上 | 是否需要通话分钟 |
| 年龄 | 年龄要求 | 部分卡有年龄限制 |
| 选号 | 能选号/支持选号 | 是否需要选号功能 |
| 照片 | 不要照片/免照片 | 是否排除需传照片的卡 |

### 第三步：筛选匹配卡片

使用 `scripts/scraper.py` 中的 `filter_cards()` 函数逻辑：

1. 按运营商过滤
2. 按月租上限过滤
3. 按通用流量下限过滤
4. 按通话分钟过滤
5. 按省份/城市匹配
6. 按年龄兼容性检查
7. 按选号/照片需求过滤

**排序规则**：主推产品优先 → 按销量从高到低

### 第四步：格式化输出（表格形式）

推荐卡片必须以 **Markdown 表格** 形式输出，调用 `scripts/scraper.py` 中的 `format_cards_table(cards)` 函数生成：

```markdown
| # | 卡片名称 | 运营商 | 月租 | 通用流量 | 通话 | 年龄要求 | 适用地区 | 已办理 | 办理 |
|---|---------|--------|------|---------|------|---------|---------|--------|------|
| 1 | 🔥卡名 | 电信 | ¥29 | 200G | 100分钟 | 18-60岁 | 仅发广东 | 3,440,112人 | [办理](url) |
```

表格列说明：
- **#**：序号
- **卡片名称**：完整名称，主推产品前加 🔥
- **运营商**：电信/联通/移动/广电
- **月租**：¥金额
- **通用流量**：XG
- **通话**：X分钟
- **年龄要求**：XX-XX岁
- **适用地区**：从关键词中提取（"仅发广东""全国可发"等）
- **已办理**：销量人数

## 筛选逻辑详解

### 省份匹配

卡片的省份限制信息分布在 `name` 和 `keywords` 字段中：

- "仅发广东"/"仅限广东"/"广东全省均可配送" → 仅广东用户
- "发全国"/"全国可发"/"全国发货" → 全国用户
- "仅发广州"/"仅发广州 东莞" → 特定城市
- "仅发广西 广东 山东 四川 重庆 陕西 浙江 江西 湖北 吉林 湖南" → 多省

匹配策略：
1. 用户指定省份 → 优先匹配"仅发该省"和"全国可发"的卡
2. 用户说"全国" → 只返回"全国可发"的卡
3. 用户未指定省份 → 展示所有卡片，但标注适用地区

### 年龄匹配

从 `age` 字段解析年龄范围（如 `年龄：18-65周岁`），检查用户年龄是否在范围内。如果用户未提供年龄，默认展示所有卡片但标注年龄要求。

### 排序优先级

1. `zhutui=true`（主推产品）→ 排最前
2. 按 `sales` 销量从高到低
3. 同等销量按价格从低到高

## 注意事项

- 实时抓取优先，确保用户拿到最新卡片数据
- agent-browser 是唯一可靠的实时数据获取方式（API 直连不可用）
- 抓取失败3次才回退缓存，并必须告知用户数据非最新
- 抓取成功后，自动更新本地缓存（写入 assets/cards_cache.json），下次兜底时缓存也是较新的
- 卡片数据可能随时更新（下架、价格变动等），每次触发技能都应优先尝试实时抓取
- 用户办理前需自行确认套餐详情，本技能仅做推荐参考
